Stroke Risk Prediction System

AI-driven web application for real-time stroke risk assessment using health and demographic data. Combines a pre-trained Linear Discriminant Analysis (LDA) model with SMOTE balancing for clinical-grade predictions. Responsive frontend for healthcare professionals and individuals, Flask-powered backend API, and Dockerized for easy deployment.

Detailed Overview

The Stroke Risk Prediction System is an AI-driven web application designed to assess an individual’s likelihood of suffering a stroke based on health and demographic inputs. By combining a pre-trained Linear Discriminant Analysis (LDA) model with SMOTE balancing, the system delivers clinical-grade risk predictions in real time. Its responsive, intuitive frontend allows both healthcare professionals and individuals to enter patient data, submit it to a Flask-powered backend API, and instantly view probability scores alongside clear risk classifications. The project is containerized with Docker for easy deployment and is currently live as a demo at the provided URL.

Key Features

AI-Powered Predictions
Uses an LDA model with SVD solver for classification
Balances the highly skewed dataset (19.5:1 non-stroke to stroke) via SMOTE
5-fold stratified cross-validation with 3 repeats yields an AUC-ROC of ~0.84

Real-Time, Clinical-Grade Accuracy
Test set AUC-ROC of 0.843; high recall (82%) prioritizes sensitivity
Detailed probability breakdown for “stroke” vs. “no stroke”
Confidence score accompanying each prediction

Modern, Responsive UI/UX
Built with React, TypeScript, shadcn/ui, and Tailwind CSS
Mobile-first, accessible design (ARIA labels, keyboard navigation)
Real-time form validation, loading states, and polished animations

Robust Backend & API
Flask RESTful API serving predictions and model info
Input sanitization and comprehensive error handling
Rate limiting (50 requests/hour/IP), security headers, health checks

End-to-End Containerization
Docker and Docker Compose configurations for both frontend and backend
Nginx for request routing (when deployed in production)
Gunicorn to serve the Flask app in production environments

Current Limitations

Class Imbalance Impact
Precision is low (≈13%) due to the rare occurrence of stroke cases, leading to false positives. SMOTE helps but does not fully resolve imbalance.

Feature Set Constraints
Limited to 10 static features; omits longitudinal trends or additional clinical biomarkers that could improve predictive power.

Generalization Risks
Trained on one dataset’s demographic distribution; may underperform for underrepresented age groups or ethnicities.

Technical & Deployment Gaps
Docker deployment requires optimization for production scaling.
CI/CD pipeline and automated testing suite are not yet implemented.
Rate limiting is basic; more granular or adaptive throttling could be added.

Planned Enhancements

Short-Term (v1.1 – v1.3): In the immediate future, the project aims to upgrade its predictive capabilities by experimenting with ensemble methods such as Random Forest and XGBoost, as well as exploring initial deep-learning prototypes. There will be a focus on visualizing feature importance to improve interpretability for end users. The user interface and experience will be enhanced with contextual input suggestions, inline tooltips, and the ability to track the history of past predictions per user session. Additionally, users will be able to generate PDF or printable risk assessment reports. On the production side, a full CI/CD pipeline will be established, including linting, tests, and automated deployment. Structured logging and error monitoring (e.g., Sentry) will be integrated, and comprehensive unit and integration tests will be written to cover all API endpoints.

Medium-Term (v2.0 – v2.5): The next phase will introduce advanced functionality such as user authentication, profile management, and session persistence. A trend-analysis dashboard will be developed to show risk variation over time, and the system will be enhanced to ingest data from wearable devices like continuous glucose monitors. Clinical system integration will be prioritized, with FHIR compliance for seamless EHR connectivity and automated clinical decision-support alerts. The architecture will become pluggable, supporting multiple disease-prediction models, and an A/B testing interface will be added to compare algorithm performance.

Long-Term (v3.0+): Looking further ahead, the project will focus on scalable AI innovations such as federated learning, enabling training on distributed hospital datasets without sharing sensitive data. A real-time model update pipeline will be implemented to incorporate new clinical data as it becomes available. An explainable AI dashboard will be developed for clinicians to explore the rationale behind predictions. The platform will also expand to include native mobile apps for iOS and Android, telemedicine integration for remote consultations, and full multi-language localization and internationalization, ensuring accessibility and usability for a global audience.
